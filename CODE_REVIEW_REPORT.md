# CODE REVIEW REPORT — L.U.M.U.S.-8 PURE Edition

## 結論

提示された原版は、研究用プロトタイプとしては理解しやすい一方、例外を黙殺し、未来情報を混入させ、取得失敗を強気判定や150円固定へ変換するため、実運用投入は不可（評価 **C**）です。本改善版は、候補生成・人手確認用途として **A-** を目標とします。取引所級データ、企業行動検証、完全なセクターマスター、ブローカー連携はPURE版の範囲外です。

## 重大所見と対応

| 原版の問題 | リスク | 改善版の対応 |
|---|---|---|
| `except: pass` / 空DataFrame継続 | 失敗銘柄が見えず、母集団が変わってランキングが歪む | chunk単位取得、例外ログ、全銘柄の品質レポート、候補不足警告 |
| 一括 `.ffill().bfill()` | `bfill` が未来価格を過去へ入れる。長期停止も架空の横ばいになる | `bfill` 廃止、品質ゲート後に内部ギャップを最大2日だけforward-fill |
| 400暦日で252営業日評価 | 休日・障害で12か月+除外月を確保できない | 2年取得、最低274観測を要求 |
| `BRK.B` 等の表記揺れ | Yahooで取得不能 | 米国のみ `.`→`-`、`.T` は保存する正規化関数 |
| レジーム取得失敗をBULL扱い | 障害時にリスク最大化 | `UNKNOWN` とし、保守的な助言稼働率にする |
| BEAR時に全スコアを `-3` | 順位が一切変わらず「厳格化」にならない | ランキングから分離し、稼働率は助言値としてmanifestへ記録 |
| 為替失敗時に150円固定 | USD注文額が誤る | FX不明時は米国注文を `BLOCKED_FX_MISSING` にする |
| 日本株を1株単位で計算 | 多くの東証株の売買単位と不一致 | 日本株100株、米国株1株の確認用ロット計算 |
| 空の注文表に `.set_index` | 例外停止 | 空でも安全なDataFrameを返す |
| 出力がprintのみ | 監査・再現不能 | CSV 4種、JSON manifest、構造化ログ |

## 実運用上の限界

1. yfinance は公式取引所フィードではなく、価格訂正、分割、配当、レート制限の検証が必要です。
2. S&P 500の現時点構成を使うため、過去バックテストでは生存者バイアスが発生します。
3. 日本株ユニバースは静的精鋭リストであり「600銘柄」を厳密に保証しません。名称を実数に合わせるか、承認済みマスターを別途管理すべきです。
4. セクターCSV未指定時は集中を測定できず、明示警告します。同一テーマ検知には別のテーマ分類マスターが必要です。
5. 出力は確認用で、成行・自動発注は意図的に実装していません。
