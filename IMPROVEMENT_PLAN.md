## 運用開始前（必須）

1. 承認済み日米ユニバースとセクター・テーマ分類を日付付きで管理する。
2. 公式または契約データで終値、分割、配当、通貨、売買単位を照合する。
3. 四半期ごとのウォークフォワード試験で、税・手数料・スリッページと回転率を評価する。
4. 現保有との差分、最小売買金額、指値ルール、二者承認を定義する。
5. 欠落銘柄、候補不足、UNKNOWNレジーム、セクター不明時の停止基準を運用規程にする。

## デモ実行例と出力サンプル

```bash
python momentum_screening_pure.py --demo --output-dir demo_output
```

正常時は20銘柄ずつの固定デモ母集団から品質ゲートと適格フィルターを通し、最大12銘柄を選定します。`run_manifest.json` には `mode: demo`、固定データから判定したUS/JPレジーム、助言稼働率、警告、残存キャッシュが記録されます。実際の候補と金額は `selected_portfolio.csv` と `order_review.csv` を参照してください。
